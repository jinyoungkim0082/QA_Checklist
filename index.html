STEP 1① 앱 정보 입력
좌측 사이드바에서 앱 이름, 버전, 플랫폼, 앱 유형을 설정하세요.

입력한 정보는 PDF 보고서 헤더에 자동으로 반영됩니다.
• 플랫폼 — iOS/Android 중 테스트 대상을 선택하면 해당 플랫폼 전용 항목만 포함됩니다.
• 앱 유형 — 유형에 따라 특화 테스트 항목이 자동 추가됩니다.
STEP 2② 기획서 업로드 · AI 자동 분석
상단 AI 분석 탭을 선택하면 기획서를 업로드해 AI가 테스트 항목을 자동 생성할 수 있습니다.

• 지원 형식 — PPT, PDF, 이미지, URL
• 지원 AI — Claude, Gemini, ChatGPT, Grok 중 선택
• AI 선택 후 해당 서비스의 API 키를 입력하면 분석이 시작됩니다.
STEP 3③ 테스트 카테고리 선택
테스트할 영역의 카테고리 태그를 선택하세요.

• 기본 10개 카테고리 제공 (기능·UI/UX·성능·네트워크 등)
• + 카테고리 추가로 원하는 테스트 영역을 직접 만들 수 있습니다.
• 여러 개 선택 가능하며, 선택된 카테고리만 체크리스트에 포함됩니다.
STEP 4④ 체크리스트 생성 · 저장
체크리스트 생성하기 버튼을 누르면 선택한 카테고리에 맞는 항목이 바로 생성됩니다.

• 💾 임시저장 — 현재 작업을 브라우저에 저장, 새로고침 후 자동 복원
• 📂 불러오기 — 이전에 저장한 체크리스트를 불러옵니다
STEP 5⑤ 테스트 항목 기록
생성된 각 항목에 테스트 결과를 기록합니다.

• PASS / FAIL / N/A / N/T 중 선택
• 우측 비고란에 메모 입력 가능
• ✎ 버튼으로 항목 텍스트·우선순위 편집
• ⠿ 드래그로 항목 순서 변경
• ✕ 버튼으로 불필요한 항목 삭제
STEP 6⑥ 진행률 확인 · 보고서 출력
상단 진행률 패널에서 전체 진행 현황을 실시간으로 확인합니다.

• 진행 / PASS / FAIL / N/A / N/T 수치 실시간 업데이트
• 💾 저장 — 2초마다 자동저장, 새로고침 후 자동 복원
• 🖨 인쇄 / PDF — FAIL 항목 중심 QA 보고서 출력
• 초기화 — 결과만 초기화, 설정과 항목 구조는 유지
